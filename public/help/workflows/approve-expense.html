Expense Approval Workflow

This guide explains the expense approval process for managers. Learn how to review, approve, and reject expense submissions efficiently.

Required Role

Manager or Administrator

Workflow Overview

  1. Staff submits expense via mobile portal or admin panel
  2. Expense enters pending status awaiting approval
  3. Manager reviews the expense details and receipt
  4. Manager approves or rejects the expense
  5. If approved: Card balance is deducted
  6. If rejected: Submitter is notified with reason

Accessing Pending Approvals

Method 1: Notification Badge

The notification bell in the header shows pending approvals. Click to go directly to items needing attention.

Method 2: Approvals Tab

  1. Go to Finance â†’ Petty Cash
  2. Click the Approvals tab
  3. View all pending expenses
Daily Check

Check the approval queue at least once daily. Pending items block staff from reconciling their expenses.

Processing an Approval

  1. Step 1: Select an Expense

    Click on any pending expense to view details.

  2. Step 2: Review Details

    Examine the expense carefully:

    • Amount: Is this amount reasonable for the expense?
    • Category: Is the category appropriate?
    • Description: Does it clearly explain what was purchased?
    • Date: When did the expense occur?
    • Submitter: Who submitted this?
    • Card: Which card is being charged?
  3. Step 3: Check Receipt

    Review the attached receipt:

    • Is a receipt attached?
    • Is the receipt legible?
    • Does the receipt amount match the expense?
    • Does the receipt date match?
    Missing Receipt

    If no receipt is attached and company policy requires one, reject and ask submitter to resubmit with receipt.

  4. Step 4: Make Decision

    Based on your review:

    Approve

    Click Approve if:

    • Amount is reasonable
    • Category is correct
    • Receipt is valid
    • Expense is justified

    Reject

    Click Reject if:

    • Amount is excessive
    • Receipt is missing/invalid
    • Expense is not justified
    • Policy violation
  5. Step 5: If Rejecting - Provide Reason

    When rejecting, always provide a clear, actionable reason:

    • Good: "Receipt missing - please resubmit with receipt photo"
    • Good: "Amount exceeds policy limit for this category - needs manager approval"
    • Bad: "Rejected" (no explanation)

What Happens After Your Decision

If Approved

  • Expense status changes to "Approved"
  • Amount is deducted from the card balance
  • Submitter sees approval in their history
  • Transaction is recorded for reporting

If Rejected

  • Expense status changes to "Rejected"
  • No balance change
  • Submitter sees rejection with your reason
  • They can submit a new expense if correctable

Processing Multiple Expenses

For efficiency, you can process multiple expenses:

  1. Review each expense in the queue
  2. Use keyboard shortcuts if available
  3. Process similar expenses together
  4. Check dashboard shows zero pending when done

Approval Best Practices

  • Be timely: Process approvals within 24-48 hours
  • Be consistent: Apply the same standards to all submitters
  • Be specific: When rejecting, explain exactly what's wrong
  • Be fair: Consider context and circumstances
  • Watch patterns: Note if someone repeatedly has issues
  • Escalate if needed: Unusual items may need admin review
